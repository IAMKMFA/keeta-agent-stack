## Operating principles for the future `/security` page

1. **Underclaim and overprove.** Anything in the table above marked "Not yet" or "No" should not
   ship in the marketing copy. Move it to a "Roadmap" subsection and link to the audit doc.
2. **Never paraphrase as absolutes.** Replace "the worker is the **only** signer" with "the worker
   is the canonical signer in our reference deployment; signing-class MCP tools are off by default."
   This matches the audit and avoids overclaiming.
3. **Link, don't paraphrase, the SECURITY.md sections.** The contact channels, disclosure timeline,
   and rotation procedure already live in `SECURITY.md`. The website should treat that file as the
   source of truth and link directly to anchored sections.
4. **Treat audit docs as the gate.** Before ANY new security claim ships on `apps/web`, the reviewer
   should confirm a row exists in this map and is marked "Yes" in the "Safe to claim today?" column.
   Phase 0B will add rows as new guards land.
